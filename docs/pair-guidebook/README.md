# People + AI Guidebook - Chapter Summaries

This directory contains chapter summaries and key concepts from the People + AI Guidebook from Google's PAIR (People + AI Research) team.

**Note**: These are summaries for reference. For the complete content with interactive elements, worksheets, and full examples, please visit the original guidebook.

## About the Guidebook

The People + AI Guidebook is a comprehensive resource for creating useful, responsible AI applications. It provides best practices and design patterns for human-centered AI product development.

## Chapters

The guidebook consists of six main chapters:

### 1. User Needs + Defining Success

**URL**: https://pair.withgoogle.com/chapter/user-needs/

Key topics:

- Finding the intersection of user needs & AI strengths
- Assessing automation vs. augmentation
- Designing & evaluating the reward function

### 2. Data Collection + Evaluation

**URL**: https://pair.withgoogle.com/chapter/data-collection/

Key topics:

- Planning to gather high-quality data
- Translating user needs into data needs
- Sourcing data responsibly
- Designing for labelers & labeling
- Model tuning

### 3. Mental Models

**URL**: https://pair.withgoogle.com/chapter/mental-models/

Key topics:

- Setting expectations for adaptation
- Onboarding in stages
- Planning for co-learning
- Accounting for user expectations of human-like interaction

### 4. Explainability + Trust

**URL**: https://pair.withgoogle.com/chapter/explainability-trust/

Key topics:

- Helping users calibrate their trust
- Planning for trust calibration throughout the product experience
- Optimizing for understanding
- Managing influence on user decisions

### 5. Feedback + Control

**URL**: https://pair.withgoogle.com/chapter/feedback-controls/

Key topics:

- Aligning feedback with model improvement
- Communicating value & time to impact
- Balancing control & automation

### 6. Errors + Graceful Failure

**URL**: https://pair.withgoogle.com/chapter/errors-failing/

Key topics:

- Defining "errors" & "failure"
- Identifying error sources
- Providing paths forward from failure
